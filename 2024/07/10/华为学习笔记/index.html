1. 版本控制工具Git

1.1 两种版本控制工具

1.1.1 集成式版本控制工具 CVS、SVN

客户端所有的操作基本上都要与服务器通信

强调集中管理，不适合开源项目

可以提供对版本库目录的精细化授权

1.1.2 分布式版本控制工具Git

分布式，提交操作、查看日志等都可以离线操作

数据、提交全部使用SHA1哈希，用于保证完整性

更适合开源项目的管理

支持移动办公场景

1.1.3 Git中项目的状态

三个区域：版本库（储存版本数据，也称本地仓库）、工作区（日常工作代码）、暂存区

三个状态：已提交、已修改、已暂存

PS：一个文件在git管理下，在修改之后会存入暂存区，经提交后在本地仓库该分支下形成一个新的节点版本，每一个节点版本都是对当前项目的文件的一次快照，对于未修改的文件会有索引指向之前的文件，相当于每个节点版本都是完整的项目

节点分支图：

1.2 Git常用命令

1.2.1 工程准备

git init 用于在本地目录下新建一个git项目仓库

git clone 用于克隆远端工程文件到本地磁盘

1.2.2 新增 删除 移动文件到暂存区

git add 用于将文件添加至暂存区

git rm 用于将文件从当前分支缓冲区中删除，不再受git的管理

git mv 用于移动、重命名文件

1.2.3 查看工作区

git diff 用于比较项目中任意两个版本（分支）之间的差异

git status 用于查看工作目录和暂存区的状态

1.2.4 提交更改的文件

git commit 用于将暂存区的文件改动提交到本地仓库

1.2.5 查看日志

git log 用于查看提交历史

1.2.6 推送远端仓库

git push 用于将本地仓库的分支推送到远程服务器上的对应分支

注意分支名字不识别字母大小写

1.2.7 分支管理

git branch 用于查看本地工程的所有git分支名称

git branch 也可用于新建分支

git branch -d 和 git branch -D 用于删除本地分支

删除后还要推送

git checkout 用于切换分支

git pull 用于从远端服务器中拉取某个分支的更新，再与本地指定分支进行合并

git fetch 也是用于从远端服务器中拉取某个分支的更新，但不会进行合并，由用户手动确认是否合并

1.2.8 分支合并

git merge 和 git rebase 用于从指定分支合并到当前分支

1.2.9 撤销操作

git reset 用于撤销工作区某些git add/commit操作，将工作区内容退回到历史节点

git checkout . 用于回退本地所有修改而未提交的文件内容（有风险）

2.2.2 类
  • 类的成员变量应该声明为private，常量可以声明为public
  • friend（友元函数或类）可以用在操作符重载，其他场景避免使用
  • 单参数构造函数声明为explicit
  • 在重写虚函数时应明确指定override或final
  • 只有基态析构函数是虚函数时，才能保证通过多态调用的时候调用到派生类的析构函数
  • 禁止子类的虚函数定义与基类不同的默认参数

2.2.3 变量
  • 使用const修饰不应该再进行修改的变量或对象（对于const修饰的对象，只允许调用其const成员函数）
  • 使用constexpr定义在编译时就可以被计算出的值

2.3 枚举、声明或定义

2.3.1 枚举
  • 避免定义未命名的枚举：如果无法命名枚举，那么这些常量可能不应该通过枚举组织在一起
  • 优先使用枚举类而不是普通枚举（使用时需要显示指定作用域，如enum class State{DEFAULT,OK} 使用：State：OK)
  • 避免将表达式转换为枚举类型
2.3.2 声明或定义
  • 不要声明或定义保留标识符（简言之不要有与编程规范冲突的命名，如int、error）

  • 变量被使用时才声明并初始化

  • 避免使用全局变量

  • 不要在头文件中定义非内联函数和非内联变量

  • 只能通过包含头文件的方式使用其他文件提供的接口和变量

2.4 类型转换、整数运算

2.4.1 类型转换
  • 禁止使用move函数操作const对象

2.4.2 整数运算
  • 确保整数运算不溢出、回绕
  • 只对无符号数做位运算

2.5 资源管理、异常处理

2.5.1 资源管理
  • 如果函数类型为数组或者指针，在参数传递时数组会退化为指针，数组长度信息会丢失，容易造成数组索引合法性检查错误。
  • 禁止将局部变量的地址返回到其作用域以外
  • 对于new操作产生错误处理时判断指针是否为空

3. C++现代实战知识

3.1 资源管理和基本对象的规则

3.1.1 栈对象与堆对象

栈对象：是分配在栈上的变量，它们的生命周期由其所在的作用域（scope）决定，当作用域结束时，这些对象会自动销毁。

特点：

  1. 自动管理内存：栈对象在其作用域结束时自动销毁，无需手动释放内存。
  2. 分配速度快：栈上的内存分配和释放速度非常快，因为它们遵循后进先出的顺序（LIFO）。
  3. 局限性：栈的大小通常有限，分配过大的对象可能会导致栈溢出。

堆对象：是分配在堆上的变量，它们的生命周期由程序员控制，需要手动分配和释放堆上的内存。

特点：

  1. 手动管理内存：堆对象需要通过 new 操作符分配内存，并通过 delete 操作符释放内存。如果忘记释放内存，会导致内存泄漏。
  2. 灵活性：可以在运行时动态分配任意大小的内存，更适合处理需要大量内存或生命周期不确定的对象。
  3. 分配速度较慢：堆内存分配和释放速度相对较慢，因为它们需要处理更多的复杂性（如碎片整理）。
3.1.2 RAII与值语义

RAII（Resource Acquisition Is Initialization，资源获取即初始化）：是C++中的一种管理资源的编程技术。它的核心思想是通过对象的生命周期来管理资源，如内存、文件句柄、网络连接等。在对象创建时获取资源，在对象销毁时释放资源，从而确保资源的正确释放，避免资源泄漏。

c++中默认使用值语义，即不明确使用引用或指针赋值时传递的是值，两个对象指向不同的内容，例如使用string创建对象后，将其直接赋值给另一个变量，另一个变量修改字符串后，源字符串不会受到影响。

3.1.3 对象创建方式：

① 自动变量（栈上分配）：对象在栈上创建，当其所在的作用域结束时自动销毁。这种方式不需要显式释放内存。

② 动态变量（堆上分配）：对象在堆上创建，必须显式释放内存。这种方式允许对象在函数作用域结束后仍然存在。

③ 智能指针（C++11及以后）：使用智能指针自动管理动态内存，避免手动释放内存的麻烦。

④ 临时对象：通过直接创建匿名对象，这些对象通常在表达式结束时销毁。

⑤ 拷贝构造和移动构造：通过拷贝或移动现有对象来创建新对象。

⑥ 列表初始化（C++11及以后）：使用大括号初始化对象，可以提高代码可读性。

3.2 移动语义与右值引用

左值：指内存中的一个位置，可以获取它的地址。它是一个可以出现在赋值操作符左侧的表达式。（左值引用：int&)

左值引用用于引用那些有名字的、持久的变量，这些变量可以在多个语句中访问。

右值：指不具备内存地址的临时对象。它是一个出现在赋值操作符右侧的表达式。(右值引用：int&&)

右值引用用于引用那些没有名字的临时对象，这些对象通常会在当前表达式结束时销毁。

move函数：用于将一个对象变成一个“可移动”的对象，避免不必要的复制，节省资源。（可将左值显示转换为右值引用）

forward函数：用于在模板中保持传入参数的原始类型（左值或右值），确保在模板函数中正确处理参数的类型，不管它是左值还是右值（转发引用）。

3.3 智能指针

智能指针：封装了原始指针，并负责在适当的时间自动释放内存。

① std::unique_ptr 是一个独占所有权的智能指针，意味着一个对象只能由一个 std::unique_ptr 管理。它不能被复制，但可以被移动。

② std::shared_ptr 是一个共享所有权的智能指针，多个 std::shared_ptr 可以指向同一个对象，并使用引用计数来管理对象的生存期。当最后一个 std::shared_ptr 被销毁时，指向的对象才会被释放。

③ std::weak_ptr 是一个弱引用智能指针，不能直接管理对象的生存期。它通常与 std::shared_ptr 搭配使用，用于解决循环引用问题。std::weak_ptr 不增加引用计数。

下列成员函数一般不允许抛异常（noexcept）：

  • 析构函数
  • 移动构造函数
  • 移动赋值运算符
  • 交换函数（swap）

临时对象在当前语句执行完成后即析构，除非赋值给引用变量而延长生命周期。

3.4 语言易用性改进

3.4.0 一般改进

[[fallthrough]] 有意从case语句直落（没有break/return）

[[maybe_unused]] 标记变量、参数可能不被使用

[[nodiscard]] 标记返回值不应被丢弃（强烈推荐用在返回错误码的函数上）

在基类使用virtual，在派生类使用override（覆盖了基类的成员函数），两者不同时使用

final用于声明成员函数是一个虚函数，且该虚函数不可在派生类中被覆盖

3.4.1 三种构造函数：

默认构造函数：创建对象时没有初始值。

复制构造函数：需要创建对象的副本时，避免了指针成员的浅复制问题，确保每个对象都有自己的资源。

移动构造函数：实现高效的资源管理，通过“移动”资源而不是复制来提高性能，特别是在处理动态内存、大数据结构或文件句柄等昂贵资源时。（需要右值引用）

3.5 any、optional、variant和错误处理

3.5.1 any

any可以存放任何类型，能够记住所存放的值的类型

any的适用场景：需要用同一类型存放多种不同类型对象

  • 回调函数的参数（代替void*）
  • 异质容器（如 vector 或 map<string,any> ）
3.5.2 optional

optional用于表示一个可能包含或不包含值的对象。它可以用来处理函数可能返回一个有效值或无值的情况，而不需要使用指针或特殊的返回值来表示错误。

optional的适用场景

  • 返回对象可能失败的情况（return{}）
  • 作为可选参数使用
3.5.3 variant

variant用于表示一个可以持有几种不同类型之一的对象。它类似于联合体（union），但更安全，并且具有更多功能。std::variant 可以用来替代 union 和 enum，以及一些更复杂的结构。

3.5.4 标准库的错误处理

常见的标准库异常

  • std::exception：所有标准库异常的基类。
  • std::runtime_error：表示各种运行时错误。
  • std::logic_error：表示逻辑错误。
  • std::out_of_range：表示访问超出范围的错误。
  • std::invalid_argument：表示无效参数错误。

常见错误码

标准库提供了许多预定义的错误码，可以使用std::errc枚举来引用它们。例如：

  • std::errc::invalid_argument：无效参数。
  • std::errc::io_error：I/O错误。
  • std::errc::permission_denied：权限被拒绝。
  • std::errc::timed_out：操作超时。

**std::error_code**：用于报告和处理错误状态。

错误码的设置：通过std::make_error_code创建，并在函数中设置。

错误码的检查：通过检查std::error_code对象来判断是否发生了错误。
