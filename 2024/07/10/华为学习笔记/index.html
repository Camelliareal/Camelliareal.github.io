1. 版本控制工具Git

1.1 两种版本控制工具

1.1.1 集成式版本控制工具 CVS、SVN

客户端所有的操作基本上都要与服务器通信

强调集中管理，不适合开源项目

可以提供对版本库目录的精细化授权

1.1.2 分布式版本控制工具Git

分布式，提交操作、查看日志等都可以离线操作

数据、提交全部使用SHA1哈希，用于保证完整性

更适合开源项目的管理

支持移动办公场景

1.1.3 Git中项目的状态

三个区域：版本库（储存版本数据，也称本地仓库）、工作区（日常工作代码）、暂存区

三个状态：已提交、已修改、已暂存

PS：一个文件在git管理下，在修改之后会存入暂存区，经提交后在本地仓库该分支下形成一个新的节点版本，每一个节点版本都是对当前项目的文件的一次快照，对于未修改的文件会有索引指向之前的文件，相当于每个节点版本都是完整的项目

1.2 Git常用命令

1.2.1 工程准备

git init 用于在本地目录下新建一个git项目仓库

git clone 用于克隆远端工程文件到本地磁盘

1.2.2 新增 删除 移动文件到暂存区

git add 用于将文件添加至暂存区

git rm 用于将文件从当前分支缓冲区中删除，不再受git的管理

git mv 用于移动、重命名文件

1.2.3 查看工作区

git diff 用于比较项目中任意两个版本（分支）之间的差异

git status 用于查看工作目录和暂存区的状态

1.2.4 提交更改的文件

git commit 用于将暂存区的文件改动提交到本地仓库

1.2.5 查看日志

git log 用于查看提交历史

1.2.6 推送远端仓库

git push 用于将本地仓库的分支推送到远程服务器上的对应分支

注意分支名字不识别字母大小写

1.2.7 分支管理

git branch 用于查看本地工程的所有git分支名称

git branch 也可用于新建分支

git branch -d 和 git branch -D 用于删除本地分支

删除后还要推送

git checkout 用于切换分支

git pull 用于从远端服务器中拉取某个分支的更新，再与本地指定分支进行合并

git fetch 也是用于从远端服务器中拉取某个分支的更新，但不会进行合并，由用户手动确认是否合并

1.2.8 分支合并

git merge 和 git rebase 用于从指定分支合并到当前分支

1.2.9 撤销操作

git reset 用于撤销工作区某些git add/commit操作，将工作区内容退回到历史节点

git checkout . 用于回退本地所有修改而未提交的文件内容（有风险）
